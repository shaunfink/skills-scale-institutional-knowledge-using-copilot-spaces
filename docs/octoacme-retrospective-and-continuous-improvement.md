# OctoAcme — Retrospective & Continuous Improvement

## Purpose
Capture learnings and convert them into actionable improvements.

## When
After each sprint, release, or important milestone. Also after incidents.

## Structure
- What went well
- What could be improved
- Action items (owner, due date)
- Follow-up on previous action items

## Retrospective Preparation Checklist

### Before the Retrospective
- [ ] Schedule the meeting with all team members
- [ ] Review previous retrospective action items and their status
- [ ] Gather relevant data (sprint metrics, incident reports, team velocity)
- [ ] Prepare the retrospective format/framework (e.g., Start/Stop/Continue, 4Ls)
- [ ] Set up collaborative tools (virtual board, anonymous feedback tool if needed)
- [ ] Remind team to come prepared with observations and ideas

### Facilitator Preparation
- [ ] Review sprint/project events and milestones
- [ ] Identify potential discussion topics based on observed challenges
- [ ] Prepare ice-breaker or opening activity to set positive tone
- [ ] Plan time allocation for each section
- [ ] Have action item template ready

## Running a Retrospective
- Timebox: 45–75 minutes depending on team size
- Use an anonymous idea board if needed to encourage candor
- Prioritize 2–3 top action items to avoid overload
- **Scrum Master** typically facilitates, ensuring psychological safety and balanced participation

## Tracking Improvements
- Add action items to the project backlog or issues with clear owners and timelines
- Review outstanding actions in the weekly PM sync
